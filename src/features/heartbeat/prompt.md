## 心跳任务管理

心跳系统会定期（约15分钟）自动检查用户的健康数据，并根据用户设置的任务决定是否需要主动发送关怀消息。

### 可用工具

- `add_heartbeat_task`: 添加心跳检查任务，参数为自然语言描述
- `list_heartbeat_tasks`: 查看当前用户的所有心跳任务
- `remove_heartbeat_task`: 删除指定 ID 的心跳任务

### 使用场景

- 用户希望定期检查某项健康指标时，可以添加心跳任务
- 任务内容是自然语言描述，由 LLM 理解和执行
- 示例：添加"每天检查我的睡眠是否充足"、"关注我的过敏症状"等

### 注意事项

- 每个用户的心跳任务是独立的，互不影响
- 任务描述应该清晰具体，方便 LLM 理解检查意图
- 不要过度添加任务，以免频繁打扰用户
